# Мюонография ядерных эмульсий — обработка данных с ML

> **Тема:** Применение методов глубокого обучения для обработки мюонографических данных ядерно-эмульсионного детектора

Репозиторий содержит скрипты разведочного анализа и поиска аномалий для базы данных угловых распределений мюонных треков, а также инфраструктуру для дальнейшей разработки ML-моделей.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/fantas-coder/MuonAnalysis
cd muon-tomography
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Подключить данные

Данные хранятся прямо в репозитории.

```python
# config.py
DATA_ROOT = Path("/path/to/your/data")
```

Быстро проверить, что всё подключено правильно:

```bash
python config.py
```

Ожидаемый вывод:


### 4. Запустить скрипты

```bash
# Разведочный анализ — 7 графиков
python muon_analysis.py

# Анализ аномалий — проверка качества данных + 1 дашборд
python muon_anomaly_analysis.py
```

Все графики сохраняются в папку `figures/`.

## Описание данных

| Параметр | Значение |
|---|---|
| Детекторов всего | 18 |
| Рабочих детекторов | 15 (исключены №8, №15, №16) |
| Уровни качества трека | npl3, npl4, npl5, npl6 |
| Варианты биннинга | 1.0°, 1.5°, 2.0°, 2.5° |
| Рабочий диапазон θ | 58° – 90° |
| Рекомендуемая конфигурация для ML | npl4 / 2.0° |
| Суммарно треков (npl4, 2.0°) | ~1.76M |

### Формат файлов

**`Tracks_DistrOutput_N.dat`** — основные данные, столбцы:
```
θ (°)    φ (°)    N_tracks
60.0     90.0     460.64
60.0     92.0     307.08
...
```

**`EffCorFile_Tracks.dat`** — поправка на эффективность, столбцы:
```
θ_D(°)  θ_D(рад)  N_треков_калибр  поправка
0.25    0.00436    722              0.689
...
```
> !!! Поправка надёжна только при θ_D ≤ 30° (соответствует θ_Земля ≥ 60°).

**`Input.txt`** — позиции детекторов в единой СК:
```
# номер  X(м)    Y(м)     высота(м)  угол_поворота(°)
1        113.3   1020.7   145.2      269.7
...
```

**`Detectors.txt`** — позиции в локальной СК ((0,0) = детектор №9):
```
# номер  X(м)    Y(м)    высота(м)
1        -45.5   27.1    74.22
...
```

## Использование config.py в своих скриптах

`config.py` содержит все общие настройки и функции. Просто импортируй нужное:

```python
from config import (
    DATA_ROOT, OUTPUT_DIR,
    GOOD_DETS, THETA_MIN, THETA_MAX,
    load_tracks, sum_all_detectors, make_grid,
)

# Загрузить данные детектора №4, npl4, биннинг 2.0°
data = load_tracks("npl4", "2.0Grad", det=4)

# Суммировать по всем рабочим детекторам
total = sum_all_detectors("npl4", "2.0Grad")

# Построить 2D-сетку для передачи в модель
phi_bins, theta_bins, grid = make_grid(total)
# grid.shape → (17, 180) для биннинга 2.0° в рабочем диапазоне
```

Если нужно поменять путь к данным **без редактирования файла** (например, в ноутбуке):

```python
import config
config.DATA_ROOT = Path("/другой/путь")
```

## Известные особенности данных

| Проблема | Детали |
|---|---|
| **Детектор №8** | Ноль треков во всех конфигурациях — технический сбой |
| **Детекторы №15, №16** | Аномально мало треков — исключены |
| **npl3** | Неполный архив: только биннинг 1.0°, нет служебных файлов |
| **npl6 / 1.0°** | Аномальный формат φ (только 2 значения вместо 360) |
| **θ < 58°** | Нулевые бины — геометрическое ограничение установки, не дефект |
| **~94–99% бинов = 0** | Норма для эмульсионного детектора |
